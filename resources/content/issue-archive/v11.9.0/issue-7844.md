# [Docs] Re-position Neo.mjs as "Agent Operating System" in Schema and Metadata

**Objective**
Update the project's documentation, metadata, and SEO positioning to reflect the strategic shift to "Agent Operating System" introduced in v11.8.0.

**Context**
Release v11.8.0 introduced the Standalone AI SDK and "Code Execution" capabilities, moving Neo.mjs beyond a simple framework into a true OS for autonomous agents. The current documentation (Schema.org metadata, `package.json` description) still reflects the previous "AI-Native Platform" positioning.

**Tasks**
1.  **Update `package.json`:**
    -   Revise `description` to highlight "Agent Operating System" and "Safe AI SDK".
    -   Add keywords: `agent-operating-system`, `code-execution-pattern`, `typescript-without-typescript`, etc.
2.  **Update Portal Schema (`apps/portal/index.html`):**
    -   Replace "AI-Native Platform" schema with "Agent Operating System".
    -   Add FAQs covering Code Execution, Self-Healing, and Runtime Type Safety.
3.  **Release Notes:** Ensure v11.8.0 notes link to Anthropic's context and explain the "TypeScript without TypeScript" innovation.

**Deliverables**
-   Updated `package.json`
-   Updated `apps/portal/index.html`
-   Verified `v11.8.0` release notes

**Strategic Value**
This re-positioning captures the emerging "Agentic" market and differentiates Neo.mjs from traditional UI frameworks by claiming the "Agent OS" category.
