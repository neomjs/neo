**Other information:**

This PR introduces a dedicated setup guide for users on **x64 Ubuntu systems** who encounter ChromaDB binding issues during AI server setup.
It documents the correct environment configuration using **Windows Subsystem for Linux (WSL)** to ensure full compatibility and prevent installation failures.
Closes #7596 

### **✅ What Was Added**

- New documentation file: [.github/setup-x64-wsl.md](https://github.com/Alachi24/neo/blob/docs/setup-x64-wsl-guide/.github/SETUP_X64-WSL_GUIDE.md)
- Step-by-step instructions for enabling virtualization, installing Ubuntu via WSL, and configuring VS Code Remote – WSL
- Notes on common pitfalls (missing virtualization, skipped Ubuntu user setup, etc.)

### **🎯 Impact**

This update ensures smoother onboarding for contributors using x64 systems by:

- Resolving the `chromadb-js-bindings-linux-x64-gnu` module error
- Providing a reliable WSL-based workaround for ChromaDB installation
- Enhancing documentation clarity for cross-architecture setup support

> 2.  **Filename:** Rename `.github/SETUP_X64-WSL_GUIDE.md` to `.github/AI_TOOLING_WSL_SETUP.md`.

> 
> This reframing ensures the guide is accurate and helps the right audience without creating unnecessary barriers for others. The content of the guide is great, it just needs to be presented in the correct context.
